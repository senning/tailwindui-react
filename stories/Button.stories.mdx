## Mode

Buttons support a few modes, with styling defined in the theme. Mode is `primary` by default. 

Since `mode` just controls colours in the default theme, **mode will be ignored if you set the `colour` prop**.

You can add more modes in the theme. For example, to add an `alert` mode:

```
//In your (optional) theme file, /theme.js
import { DefaultTheme } from 'tailwindui-react'
let theme = DefaultTheme;
theme.default.buttons.alert = ["bg-yellow-800", "text-yellow-100", ...]
export theme;

//In your project root (i.e. /App.js)
import { ThemeProvider } from 'tailwindui-react';
import theme from 'theme';

<ThemeProvider initialState={theme}>
  <Button mode="alert">
</ThemeProvider>
```

## Colour

You can use any colour in your Tailwind theme in `colour`. By default, `Button` will use the 800 version as the background colour and the 100 version as the text colour. So if you use `colour="orange"`, the button will have the classes `bg-orange-800 text-orange-100`. You can flip this around with the `appearance` property.

## Style

Works with the Colour property. With `solid` (the default) the 800 version of the colour is used as the background and the 100 version is used as the text colour. `inverted` flips this, `outlined` adds a border to `invert` and `text` uses the 800 version of the colour for the text only.

## Icon

You can add icons to buttons. `icon` and `iconLeft` add the icon to the left. `iconTop`, `iconRight` and `iconBottom` add icons to the top, right and bottom respectively. This uses the Icon component, which supports XXXXXXXXXXX.

For more control, you can also set `Icon` as a child of `Button`.
